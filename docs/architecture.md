# Архитектура SecureLinux-NG

## Цели
- полное соблюдение требований и рекомендаций ФСТЭК;
- прозрачная трассировка требований к коду;
- управляемое применение изменений;
- предсказуемый откат;
- контроль совместимости до применения.

## Основные режимы
- проверка
- применение
- откат
- dry-run
- отчёт покрытия

## Принципы
- каждый блок кода должен иметь привязку к пункту стандарта;
- каждое изменение должно быть либо обратимым, либо явно помечено как необратимое;
- все изменения должны фиксироваться в manifest/report;
- совместимость должна проверяться до применения.
