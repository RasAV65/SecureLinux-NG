# Модель отката

## Цель
Любое изменение должно иметь понятный сценарий отката.

## Требования
- manifest должен хранить все созданные файлы;
- manifest должен хранить все созданные группы;
- manifest должен хранить резервные копии изменённых файлов;
- отчёт должен показывать, что можно откатить, а что нельзя;
- необратимые изменения должны быть явно помечены.

## Минимальные сущности manifest
- backups
- created_files
- created_groups
- sysctl_configs
- systemd_units
- grub_backups
- apply_report
